# Описание
Утилита **docgen** - это инструмент, позволяющий в автономном режиме поддерживать актуальное состояние документации с помощью собственной агентной системы. Инструмент имеет прямой доступ к проекту через git-репозиторий, в котором содержится исходный код приложения (и не только), на основе чего и поддерживается документация.  
Для корректной работы утилиты желательно, чтобы анализируемый репозиторий уже содержал в себе хоть какую-то документацию для первоначального анализа.  

# Использование
[!IMPORTANT]  
Пока что доступно только для Linux.  

Клонируем репозиторий и переходим в него:
```bash
git clone https://github.com/daisccho/docgen.git
cd docgen
```
Создаём виртуальное окружение и активируем его:
```bash
python3 -m venv .venv
source .venv/bin/activate
```
Обновляем pip и устанавливаем зависимости:
```bash
pip install --upgrade pip
pip install -e .
```
[!NOTE]  
pip install с флагом -e позволит вызвать команду docgen из любого места в вашей ОС.

# Доступные команды
- **docgen init [OPTIONS]**  
Инициализация проекта, генерация **.docgen.yaml**. Следует вызывать в отдельно созданной папке (не в папке репозитория). Здесь будут храниться все документации по отдельным коммитам, логи агента и **SUMMARY.md**, генерируемый агентом в ходе анализа проекта.  
Опции:  
    * [required] --repo TEXT - URL git-репозитория;  
    * --token-env TEXT - Токен доступа к репозиторию или имя переменной окружения, содержащей его;  
    * --api-key TEXT - API-ключ LLM или имя переменной окружения, содержащей его;    
    * --base-url TEXT - Base URL для OpenAI-совместимого API;  
    * --model TEXT - Используемая модель.  

- **docgen snapshot [OPTIONS]**  
Создаёт доверительную версию документации на основе всех md-файлов в проекте с сохранением их местоположения от корневой директории.  
Опции:  
    * -r, --ref TEXT - хэш/тег коммита, по которому необходимо сделать снапшот; 
    * -c, --check - режим полного аудита для проверки актуальности документации;  
    * -i, --iterations INTEGER - ручное указание максимального количества ходов при вызове инструментов; 
    * -l, --log - логирование действий; 
    * -v, --verbose - подробный вывод в консоли.

- **docgen watch [OPTIONS]**  
Запуск инструмента в режиме демона, в котором приложение через указанный промежуток времени будет проверять репозиторий на наличие новых коммитов и поддерживать документацию в стабильном состоянии.  
Опции:  
    * -b, --branch TEXT - название ветки, за которой демон будет следить; 
    * -t, --interval INTEGER - интервал проверки ветки на новые коммиты;  
    * -i, --iterations INTEGER - ручное указание максимального количества ходов при вызове инструментов; 
    * -l, --log - логирование действий; 
    * -v, --verbose - подробный вывод в консоли. 

- **docgen versions**  
Просмотр имеющихся версий документации проекта. Актуальная документация, прошедшая ревизию агента, отмечена звездочкой.

# Инструменты ИИ-агента

- **terminal**  
Реализует функционал полноценного использования терминала. Имеет встроенный черный список, мешающий агенту выполнять особо опасные команды.

- **read_summary**  
Читает файл SUMMARY.md, содержащий в себе сводку по проекту. 

- **write_summary**  
Создаёт/изменяет SUMMARY.md по итогам работы агента с репозиторием.

- **collect_changelog**  
Собирает из репозитория файлы, содержащие описание изменений, дополняя их комментариями из коммитов.

# Файл конфигурации
После запуска команды **docgen init** в cwd создается файл **.docgen.yaml**, который имеющий следующую структуру:
```yaml
config:
  git_repo: https://github.com/blank/repo
  access_token: null # нужен только в случае, если репозиторий приватный
  llm_provider: openai
  llm_model: gpt-4o
  llm_api_key: null  # или OPENAI_API_KEY в env
  llm_base_url: null # для OpenAI-совместимых провайдеров
  project_name: default  # имя проекта
  max_turns: 100
last_documented_commit: null
initial_commit: null
```

# Генерация снапшота

**Снапшот** - это сохраненная вручную документация, которая служит отправной точкой (в случае первого создания) для ИИ-агента в последующем поддержании документации проекта.  
Для генерации снапшота в папке проекта необходимо запустить следующую команду:
```bash
docgen snapshot
```
Без указания параметров снапшот будет сделан с последнего коммита. В конфиге переменные **last_documented_commit** и **initial_commit** будут изменены, чтобы отразить текущее состояние проекта.  
При последующий вызовах **snapshot** будут генерироваться и другие версии документации, но в данном случае изменяться будет только **last_documented_commit**. ИИ-агент при запуске в виде демона будет изменять документацию, ссылаясь на последний задукоментированный коммит.

# Автоматическое поддержание документации
При запущенном процессе с помощью **docgen watch** каждые **N** минут ИИ-агент будет генерировать новую документацию в отдельной папке, названной по хэшу коммита. Работа агента заключается в генерации в рабочей директории следующих файлов:
```yaml
docgen_project_dir:
    SUMMARY.md
    logs/:
        snapshot_20260708_135047.log
        watch_20260708_135137.log
    908973e/:
        docs/:
            README.md
            Installation.md
            Webhooks.md
            ...
    82f5089/:   # условная стартовая документация, на основе неё 
        docs/:  # агент в дальнейшем генерирует все остальное
            README.md
            Installation.md
```
Артефакты после работы утилиты:
- **SUMMARY.md**  
Файл, генерируемый агентом, в котором содержится сводка по проекту, над которым он работает. Агент читает этот файл каждый раз, когда начинает процедуру обновления документации, чтобы поддерживать свои знания о проекте. Данный файл обновляется в конце рабочего цикла **watch** перед уходом в сон.

- **snapshot/watch_date_time.log**  
Лог-файлы, в которых записываются действия агента от момента запуска команды до её завершения. Все логи храняться в директории **logs/** в корневой папке.

- Различные **.md**-файлы, в которых и содержится документация. Старая обновляется на основе новой информации, новая (не фигурировавшая нигде ранее) добавляется.

ИИ-агент в процессе своей работы самостоятельно просматривает исходную документацию, комментарии к коммитам, измененные файлы и т. д. и принимает решение, какие файлы необходимо изменить, какие добавить, какие убрать и прочее.
