USER GUIDE

Как пользоваться Docgen

1. Общие настройки

Пользователь с признаком staff открывает вкладку «Настройки» и задаёт модель, Base URL, максимум ходов агента, API-ключ LLM и GitHub-токен. Эти значения применяются ко всем проектам. Обычным пользователям секреты не показываются.

2. Создание проекта

Нажмите «Добавить проект», укажите название, URL репозитория и интервал наблюдения. Идентификатор создаётся автоматически и используется как имя рабочего каталога. После сохранения дождитесь завершения инициализации.

3. Snapshot

Docgen работает только с релизными Git-тегами. На странице проекта укажите существующий тег релиза или оставьте поле пустым для последнего релиза. Если в репозитории нет тегов, snapshot не создаётся. Флаг полного аудита включает проверку Markdown-файлов через LLM. Нажмите «Поставить в очередь» и следите за заданием.

4. Наблюдение

Режим наблюдения периодически проверяет появление новых Git-тегов с интервалом, заданным в проекте. Активный процесс можно открыть в мониторе или остановить со страницы проекта.

5. Задания и ошибки

Worker последовательно выполняет задания. На странице задания доступны статус, лог, ошибка и рекомендация. После исправления причины ошибочное или отменённое задание можно повторно поставить в очередь с тем же ID.

6. Версии

Завершённые снимки отображаются на странице проекта по тегу релиза и SHA коммита. Откройте версию, чтобы просмотреть дерево Markdown-документов.

Django Admin

Это служебная панель для superuser и пользователей staff: она предназначена для управления пользователями и ручного просмотра записей. Для обычной работы используйте основные страницы Docgen.
